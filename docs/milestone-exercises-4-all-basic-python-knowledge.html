Overview

Each exercise builds upon the previous one, so complete them in order.

Exercise 1: Network Device Inventory Manager (Easy)

Create a comprehensive network device management system that reads device information from a file and provides various analysis functions.

Requirements:

  1. Function: load_network_devices(filename)
  2. Read the network_devices.txt file
  3. Return a list of dictionaries, where each dictionary represents a device with keys: "name", "ip", "mac", "status"

  4. Handle file not found errors gracefully

  5. Function: display_device_summary(devices)

  6. Print a formatted summary showing:

    • Total number of devices
    • Number of active devices
    • Number of inactive devices
    • List all device names with their status

  7. Function: find_devices_by_status(devices, status)

  8. Return a list of devices that match the given status ("active" or "inactive")

  9. The status parameter should be case-insensitive

  10. Function: get_device_by_ip(devices, ip_address)

  11. Return the device dictionary that matches the given IP address

  12. Return None if no device is found

  13. Function: validate_ip_format(ip_address)

  14. Check if the IP address follows the basic IPv4 format (xxx.xxx.xxx.xxx)
  15. Return True if valid, False otherwise

  16. Use string methods and basic validation (don't worry about valid IP ranges)

  17. Function: create_device_report(devices, output_filename)

  18. Create a report file that lists all devices in a formatted way
  19. Include device name, IP, MAC address, and status
  20. Add a header with the total count of devices

Main Program:

Write a main section that:
- Loads the devices from the file
- Displays the device summary
- Asks the user to enter an IP address and shows the corresponding device info
- Creates a device report file called "device_report.txt"

Exercise 2: Firewall Log Analyzer (Medium)

Building on Exercise 1, create a firewall log analysis system that processes security logs and integrates with your device inventory.

Requirements:

  1. Function: load_firewall_logs(filename)
  2. Read the firewall_logs.txt file
  3. Return a list of dictionaries with keys: "timestamp", "ip", "port", "action", "protocol"

  4. Handle file errors appropriately

  5. Function: parse_log_entry(log_line)

  6. Take a single line from the log file and convert it to a dictionary

  7. Handle any malformed lines by returning None

  8. Function: filter_logs_by_action(logs, action)

  9. Return all log entries that match the specified action ("ALLOW" or "DENY")

  10. Make the search case-insensitive

  11. Function: get_top_ports(logs, limit=5)

  12. Analyze all log entries and return the most frequently accessed ports
  13. Return a list of tuples: [(port, count), (port, count), ...]

  14. Sort by count in descending order

  15. Function: analyze_ip_activity(logs, devices)

  16. Cross-reference firewall logs with your device inventory

  17. Return a dictionary where keys are IP addresses and values are dictionaries containing:

    • "device_name": name from device inventory (or "Unknown" if not found)
    • "total_requests": total number of log entries for this IP
    • "allowed_requests": number of ALLOW entries
    • "denied_requests": number of DENY entries

  18. Function: detect_suspicious_activity(logs, max_denied_requests=3)

  19. Find IP addresses that have more than max_denied_requests DENY entries
  20. Return a list of dictionaries with IP address and deny count

  21. Sort by deny count in descending order

  22. Function: generate_security_report(logs, devices, output_filename)

  23. Create a comprehensive security report file
  24. Include:
    • Summary statistics (total logs, allowed vs denied)
    • Top 5 most accessed ports
    • IP activity analysis
    • List of suspicious IPs
    • Timestamp of the report

Main Program:

Write a main section that:
- Loads both device inventory and firewall logs
- Displays summary statistics
- Shows the top 5 ports
- Lists any suspicious IP addresses
- Generates a security report file

Exercise 3: Network Security Dashboard (Hard)

Create an advanced network monitoring system that combines everything from previous exercises and adds real-time monitoring capabilities, network topology mapping, and advanced security analysis.

Requirements:

  1. Function: NetworkMonitor class or set of advanced functions for:

a. load_all_data(device_file, log_file, topology_file=None)
- Load and validate all data sources
- Return a comprehensive data structure containing all network information
- Handle missing files gracefully

b. create_network_topology(devices)
- Create a network topology representation
- Group devices by IP subnet (e.g., 192.168.1.x, 10.0.0.x)
- Return a dictionary where keys are subnets and values are lists of devices

c. calculate_network_health(devices, logs)
- Calculate an overall network health score (0-100)
- Consider factors like:
- Percentage of active devices
- Ratio of allowed vs denied requests
- Number of unique protocols in use
- Distribution of traffic across devices

d. detect_anomalies(logs, devices)
- Advanced anomaly detection combining multiple factors:
- Unusual port access patterns
- High volume of requests from single IPs
- Access attempts to inactive devices
- Unusual time patterns (if you want to add time analysis)
- Return a detailed report of anomalies found

e. generate_network_map(topology, output_filename)
- Create a text-based network map file showing:
- Network subnets
- Devices in each subnet
- Device status and types
- Connection indicators

f. create_dashboard_summary(devices, logs, topology, health_score, anomalies)
- Generate a comprehensive dashboard summary
- Include visual elements using ASCII art or simple text formatting
- Show key metrics, alerts, and recommendations

  1. Advanced Analysis Functions:

a. analyze_protocol_usage(logs)
- Analyze which protocols are most/least used
- Calculate percentage distribution
- Identify unused standard ports

b. device_communication_matrix(logs, devices)
- Create a matrix showing which devices communicate with external IPs
- Identify devices that never appear in logs (potential security risk)

c. time_based_analysis(logs)
- Analyze traffic patterns by hour (extract hour from timestamp)
- Identify peak usage times
- Detect unusual activity outside normal hours

  1. Configuration and Settings:

a. load_monitoring_config(config_file="monitor_config.txt")
- Load monitoring thresholds and settings from a configuration file
- Handle default values if config file doesn't exist
- Config should include settings like:
- Maximum allowed denied requests per IP
- Health score thresholds
- Alert thresholds for various metrics

b. save_monitoring_state(data, filename="network_state.txt")
- Save current network state to a file for comparison in future runs
- Include timestamps and key metrics

Advanced File Operations:

  • Create a configuration file with monitoring parameters
  • Implement backup functionality for all generated reports
  • Add the ability to archive old log files
  • Create multiple output formats (summary, detailed, CSV export)

Main Program:

Create a comprehensive main program that:
1. Loads all configuration and data files
2. Performs complete network analysis
3. Displays an interactive dashboard summary
4. Generates multiple report files
5. Saves current state for future comparison
6. Provides menu-driven options for different analysis views
7. Handles all errors gracefully with informative messages

Bonus Challenges:

  • Add input validation for all user inputs
  • Implement a simple command-line menu system
  • Add the ability to compare current analysis with previous runs
  • Create a function to export data in CSV format for external analysis
  • Implement basic data visualization using text-based charts

Testing Your Solutions

For each exercise, test your functions with:
- Valid input files
- Missing files
- Empty files
- Malformed data
- Edge cases (empty lists, invalid IP addresses, etc.)

Make sure your code handles errors gracefully and provides meaningful error messages to users.
